Section 1
üì• How to Copy Any HTML Code from GitHub

To ensure you get the functional code without extra HTML formatting from GitHub's UI, follow these steps:

  1. Navigate to the repository and click on the specific .html file you wish to copy.
  2. Locate the "Raw" button at the top right of the file preview pane.
  3. Clicking Raw will open a plain text version of the code in your browser.
  4. Press Ctrl + A (Windows) or Cmd + A (Mac) to select everything.
  5. Press Ctrl + C to copy the clean, unformatted code to your clipboard.

üí° Pro Tip:

Using the "Raw" view prevents accidental inclusion of line numbers or GitHub UI elements that could break your site's design.

Section 3
‚úèÔ∏è Editing Master Index.html Directly in GitHub

You don't need external software to update your site. GitHub's built-in editor is sufficient for quick changes.

  1. Go to your repository and click on index.html.
  2. Click the Pencil Icon (Edit this file) in the top right corner.
  3. Paste your updated code (generated in Step 2) into the editor, replacing the old code.
  4. Scroll down to the "Commit changes..." box.
  5. Enter a short description (e.g., "Added link to New Guide") and click Commit changes.

Section 4
üîÑ Renaming a File to Index.html

If you have a file named home.html or temp.html and want it to be your homepage, it must be named index.html.

The Renaming Procedure:

  1. Navigate to the file you want to rename in your GitHub repository.
  2. Click the Pencil Icon to enter edit mode.
  3. In the filename box at the top (where the current name is displayed), click inside and delete the old name.
  4. Type index.html exactly.
  5. Scroll to the bottom and click Commit changes.

‚ö†Ô∏è Critical Note:

GitHub Pages looks specifically for index.html to serve as the root. If you already have an index.html, you must rename or delete the old one first, as two files cannot share the same name.
